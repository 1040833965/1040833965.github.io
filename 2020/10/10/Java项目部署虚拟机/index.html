Java项目部署Linux

项目部署分为两种

  • 手工部署
  • shell脚本部署

一、手工部署

1.将项目打成jar包，使用maven的package

2.在虚拟机中创建存放jar包的目录

3.设置后台运行，并将日志输出到日志文件中

这里就有个重要的命令

nohup no hang up(不挂起) 用于不挂断的运行指定命令，退出终端不会受影响 语法格式：nohup Command [Arg…] [&]

参数说明：

  • Command:要执行的命令
  • Arg：一些参数，可以指定输出文件
  • &：让命令在后台运行

ex: nohup java -jar boot工程.jar &>bootLog.log & 后台运行java -jar 命令，并将日志输出到hello.log文件中

4.停止springboot程序

手动部署完成

安装maven

为用户授权

这里就要提到chmod命令 可以通过八进制数来指定权限

# 权限 rwx
7 读+写+执行 rwx
6 读+写 rw-
5 读+执行 r-x
4 只读 r–
3 写+执行 -wx
2 只写 -w-
1 只执行 –x
0 无 —

举例：chmod 777 fileName 为所有的用户赋予读写执行权限

  • 第一位数是 文件拥有者的权限
  • 第二位数是 同组用户的权限
  • 第三位数是 其他用户的权限

设置静态ip

设置静态ip是为了将虚拟机的ip固定在一个ip上，

因此我们需要设置一个静态的ip
